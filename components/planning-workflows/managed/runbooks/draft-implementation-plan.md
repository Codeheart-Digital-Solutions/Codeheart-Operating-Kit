# Draft Implementation Plan

Use this runbook to turn accepted discovery and targeted repo research into an execution-ready
implementation document.

## Procedure

1. Read the accepted discovery document when one exists.
2. Confirm the feature slug and implementation document path.
3. Gather targeted repo context and relevant runbooks.
4. Define the measurable implementation goal.
5. Record current and target state.
6. Make strategy decisions before writing tasks.
7. Record open questions as `OQ-*` with `BLOCKER: yes` or `BLOCKER: no`.
8. Draft a strict linear epic map.
9. For each epic, include outcome, scope, files touched, acceptance criteria, dependencies, tasks,
   implementation notes, and open questions.
10. End each epic with validation tasks.
11. Keep checklist items granular and concrete.
12. Add future planning for deferred work and later hardening.

## Requirements

Implementation plans must be self-contained enough for a new implementer to execute without
guessing. They must not branch into competing task paths after a strategy decision is chosen.

## Lifecycle

Create implementation docs as `Status: draft`. Move them to `Status: active` only after explicit
approval for execution. Tick checkboxes only after completed and validated work.
