# Session Ledger Maintenance

Use this runbook when curating agent memory from recent work.

## Procedure

1. Identify the current workstream and canonical docs.
2. Verify current repository state before relying on prior memory.
3. Summarize only durable state, decisions, blockers, handoffs, and references.
4. Keep raw transcript details out of curated memory.
5. Update overview order to match detailed entry order.
6. Use the status and relation labels from `reference/entry-format.md`.
7. Preserve existing consumer-owned memory entries.
8. Archive stale or completed entries only when the consumer's memory rules say to do so.

## Boundaries

Memory maintenance describes how to curate state. It does not replace canonical product docs,
planning docs, execution logs, or source control history.
