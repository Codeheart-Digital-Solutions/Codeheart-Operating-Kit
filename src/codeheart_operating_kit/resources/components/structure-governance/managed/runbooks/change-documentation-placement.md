# Change Documentation Placement

Use this runbook before adding, moving, renaming, or reclassifying documentation.

## Procedure

1. Identify the durable owner.
2. Decide whether the material is a plan, runbook, reference, router, archive, scaffold, template,
   or managed doc.
3. Check for an existing owner before adding a new document.
4. Promote duplicated durable rules to the owning reference.
5. Keep plan-scoped artifacts inside the plan bundle until they become enduring doctrine.
6. Update nearest README routers and parent indexes.
7. Record consumer impact when generated surfaces or managed routes change.
